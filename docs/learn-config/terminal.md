# mac 终端常用命令

## 场景说明

这一页收录日常最常用的 mac 终端命令，同时补充 mac 特有能力和 iTerm2 常见操作，适合刚开始使用 Terminal / iTerm2 时快速查阅。

## 命令表

| 命令              | 作用                     | 说明                                           |
| ----------------- | ------------------------ | ---------------------------------------------- |
| `pwd`             | 查看当前目录             | 先确认自己在哪个路径下                         |
| `ls`              | 列出当前目录内容         | 常配合 `ls -la` 查看隐藏文件和详细信息         |
| `cd <dir>`        | 切换目录                 | `cd ..` 返回上一级，`cd ~` 回到用户目录        |
| `mkdir <dir>`     | 创建目录                 | 常配合 `mkdir -p a/b/c` 递归创建多级目录       |
| `touch <file>`    | 创建空文件               | 也可用于刷新文件修改时间                       |
| `cp <src> <dest>` | 复制文件                 | 目录复制常用 `cp -R`                           |
| `mv <src> <dest>` | 移动或重命名             | 同路径下改名、本地文件整理都常用               |
| `rm <file>`       | 删除文件                 | 删除目录常见 `rm -r`，操作前先确认路径         |
| `open <path>`     | 用 Finder 或默认应用打开 | `open .` 打开当前目录，`open file.md` 打开文件 |
| `vim <file>`      | 在终端里编辑文件         | 适合快速改配置、记笔记和处理小文件             |
| `clear`           | 清空当前屏幕显示         | 只清屏，不会删除历史输出                       |
| `history`         | 查看命令历史             | 配合搜索可以找回之前执行过的命令               |
| `which <cmd>`     | 查看命令位置             | 适合确认命令来自系统还是自装工具               |
| `man <cmd>`       | 查看命令手册             | 想看完整参数说明时很有用                       |

## mac 常用补充

| 命令                 | 作用                     | 说明                                  |
| -------------------- | ------------------------ | ------------------------------------- | ------- |
| `open .`             | 在 Finder 中打开当前目录 | 最常用的 mac 终端联动命令             |
| `open -a <App>`      | 用指定应用打开           | 例如 `open -a "Visual Studio Code" .` |
| `pbcopy`             | 把内容写入剪贴板         | 常见用法：`pwd                        | pbcopy` |
| `pbpaste`            | 读取剪贴板内容           | 可以直接粘到终端命令流中              |
| `say "text"`         | 系统朗读文本             | 适合演示或测试系统语音                |
| `qlmanage -p <file>` | Quick Look 预览文件      | 不进入应用也能快速看文件              |

## iTerm2 常用操作

### 常用快捷操作

| 操作         | 默认快捷键              | 说明                     |
| ------------ | ----------------------- | ------------------------ |
| 新建标签页   | `Cmd + T`               | 开一个新的 shell 会话    |
| 关闭标签页   | `Cmd + W`               | 关闭当前标签页           |
| 切换标签页   | `Cmd + 数字`            | 快速跳转到指定标签页     |
| 垂直分屏     | `Cmd + D`               | 左右分栏查看多个会话     |
| 水平分屏     | `Cmd + Shift + D`       | 上下分栏处理多个任务     |
| 面板间切换   | `Cmd + Option + 方向键` | 在不同分屏之间移动焦点   |
| 搜索输出     | `Cmd + F`               | 在当前终端历史输出中搜索 |
| 清屏         | `Cmd + K`               | 比输入 `clear` 更快      |
| 打开粘贴历史 | `Cmd + Shift + H`       | 查看最近复制过的内容     |

### 常见使用场景

- 一个面板跑 `pnpm dev`，另一个面板看日志或执行 Git 命令。
- 用标签页区分不同项目，避免在同一窗口里反复 `cd`。
- 搜索终端输出时优先用 `Cmd + F`，比肉眼翻日志快很多。
- 需要把当前路径发给别人时，用 `pwd | pbcopy` 最省事。

## 常见误区 / 注意事项

- `rm -r` 和 `rm -rf` 删除前一定先确认当前路径，尤其不要在项目根目录误删。
- `touch` 只能创建空文件，不会自动写入内容。
- `open .` 是打开当前目录，不是进入当前目录；真正切目录还是用 `cd`。
- `clear` 只是清屏，历史输出通常还在；如果想彻底清空滚动历史，优先用 iTerm2 的清屏快捷操作或菜单项。
- `vim` 退出前先确认当前模式；常见流程是按 `Esc` 后输入 `:wq` 保存退出，或 `:q!` 放弃修改。
